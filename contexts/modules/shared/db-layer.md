# DB layer

> Postgres via `pg.Pool` + Kysely with a code-generated `DB` type; migrations are file-based (Kysely `FileMigrationProvider`) and applied manually; trigger-driven account balances; on-read computation for envelope balances; `safeAwait` tuples for ergonomic error handling.

## Components

- `apps/server/src/db/index.mts` — `createPGPool()` factory. Reads `ENV.DATABASE_URL`, sets `ssl: { rejectUnauthorized: false }`, and on every `connect` event runs `SET TIME ZONE '<APP_TIMEZONE>'` (+ `SET default_transaction_read_only = on` when `IS_DEMO`).
- `apps/server/src/db/kysely/index.mts` — `createQueryBuilder(pool)` returns `Kysely<DB>`. Used once via `createServices()`.
- `apps/server/src/db/kysely/types.mts` — auto-generated by `kysely-codegen`. **Never hand-edit.** Regenerate via `pnpm generate-types`.
- `apps/server/src/db/kysely/migrator.mts` — applies migrations from `./migrations` using `FileMigrationProvider`. Invoked directly via `pnpm migrate` (which calls `tsx ./src/db/kysely/migrator.mts`); `bootstrap.mts:9` has the auto-up-on-start call commented out, so migrations are manual.
- `apps/server/src/db/kysely/migration-template.mts` — empty `up`/`down` skeleton; copy when adding a migration.
- `apps/server/src/db/kysely/seed.mts` — `pnpm seed`; wipes product tables and loads ~8000 demo transactions across 5 spaces / 8 users. Refuses to run when `NODE_ENV=production`. Does NOT touch `files` / `*_attachments` / `exported_reports`.
- `apps/server/src/db/kysely/audit.mts` — entirely commented out. A scaffolded "shared audit columns" helper that was never adopted; ignore unless reviving.
- `apps/server/src/utils/safeAwait.mts` — `safeAwait<T,E>(promise) => Promise<[E, null] | [null, T]>`. Used everywhere; turns a `try/catch` into a tuple destructure.
- `apps/server/src/env.mts` — `DATABASE_URL` defaults to the docker-compose Postgres; `APP_TIMEZONE` defaults to `Asia/Dhaka` (UTC+06:00, no DST).

## Flow

### Migrations

- Filenames mix 4-digit (`0001` through `0017`) and 3-digit (`018` through `038`). Kysely sorts lexicographically so `018_` follows `0017_` correctly. **Keep using 3 digits for new migrations.**
- `pnpm migrate` → `tsx ./src/db/kysely/migrator.mts up` (or `down`) → `migrator.migrateToLatest()` (or `migrateDown()`). Results are printed as a markdown table (`migrator.mts:36-74`).
- `bootstrap.mts:9` shows that auto-migrate-on-start is intentionally disabled in dev/prod; the dev loop is `pnpm migrate` then `pnpm dev`.
- After any schema-changing migration: `pnpm generate-types` to refresh `types.mts`.

### Schema highlights (verify against current code)

- Tables for auth: `users` (0001), `tmp_users` (0002), `email_verification_codes` (0003).
- Spaces / membership / accounts: 0004 → 0009.
- Envelopes / events / categories / transactions: 0010 → 0013.
- Allocations + balances: 0014 (envelop_allocations), 0015 (envelop_balances), 0016 (plans), 0017 (plan_allocations).
- Triggers (still live): **018** creates `__sync_account_balance_from_transactions` on `transactions` (account balances are materialised). **030** rewrites the `__apply_transaction_balance_effect` function so transfer fees debit `source` by `amount + fee_amount`.
- Triggers (retired): **019** created envelope-balance triggers; **021** added the plan-balance trigger; **026** drops all of them along with the `envelop_balances` and `plan_balances` tables. After 026, envelope and plan balances are computed on read by the procedure code, not maintained by triggers. `down()` in 026 restores everything for rollback.
- Later migrations: 027 (FK on-delete restrict), 028 (`files`), 029 (`transaction_attachments`, `event_attachments`, `exported_reports`, `users.avatar_file_id`), 030 (transfer fees), 031 (category priority), 032 (envelope borrow), 033 (envelope archived), 034 (`idempotency_keys`), 035 (envelop carry policy), 036 (`reckoning_acknowledgments`), 037 (`spaces.budget_mode`), 038 (event lifecycle + estimate).
- **Budgeting simplified later — much of the above was undone.** Migration **046** drops the `plans`/`plan_allocations` subtree and adds `envelops.target_amount`/`target_date` (goals are now just `cadence='none'` envelopes with an optional target). Migration **048** (`048_simplify_budgeting`) reverses the borrow/carry/strict/reckoning machinery: it drops the borrow link (`borrowed_link_id`) + borrow rows, the typed-ledger seam (`kind`, `effective_at`), per-account allocation (`envelop_allocations.account_id`), carry-over (`envelops.carry_policy`/`carry_over`), strict mode (`spaces.budget_mode`), and the `reckoning_acknowledgments` table. It collapses `envelop_allocations` to **one absolute row per (envelope, period)** — one row per month for monthly envelopes, one lifetime row (`period_start` NULL) for rolling/goal — enforced by a `(envelop_id, period_start) NULLS NOT DISTINCT` unique index. So treat the 032/035/036/037 columns/tables above as historical: they no longer exist after 048.

### Live trigger (migration 018 + 030)

`AFTER INSERT/UPDATE/DELETE ON transactions` calls `__sync_account_balance_from_transactions` → `__apply_transaction_balance_effect(NEW/OLD, ±1)`. The function branches on `tx.type` (`income | expense | transfer | adjustment`) and upserts `account_balances` rows. After 030, transfers also deduct `tx.fee_amount` from the source account; the destination still receives the plain `amount`. Update operations call the function twice (reverse OLD, apply NEW) which makes UPDATEs idempotent vs the materialised balance.

### Timezone handling

`db/index.mts:21-32` runs `SET TIME ZONE '<APP_TIMEZONE>'` (not `SET LOCAL`) on every new pooled connection. This is the only place the zone is configured — every `DATE_TRUNC('month', NOW())`, `::date`, and `timestamptz → timestamp` cast inherits it. Don't pass zone literals in SQL; rely on the session setting.

### Transaction + safeAwait pattern (the canonical mutation shape)

Almost every multi-row mutation looks like this (`account/create.mts`, `auth/signup/initiate.mts:49`, etc.):

```ts
const [error, data] = await safeAwait(
    qb.transaction().execute(async (trx) => {
        // ... multiple inserts/updates using trx, not qb ...
        return result;
    })
);
if (error) {
    logger.error(`...: ${error}`);
    if (error instanceof TRPCError) throw error;
    throw new TRPCError({ code: "INTERNAL_SERVER_ERROR", message: "..." });
}
```

Key rules:
- Inside the transaction callback, use `trx`, never `qb` (the outer pool) — using `qb` escapes the transaction and silently breaks atomicity.
- Re-throw `TRPCError` as-is (so `BAD_REQUEST`, `NOT_FOUND`, etc. pass through). Wrap anything else as `INTERNAL_SERVER_ERROR`.
- `safeAwait` returns `[error, null]` on failure and `[null, data]` on success — the tuple is non-discriminated; either always check `error` first or destructure carefully.

### Idempotency cache (migration 034)

Generic `idempotency_keys(key PK, user_id, operation, response jsonb, expires_at)`. Used by mutations that touch multiple tables in one call (transfer) via `apps/server/src/utils/withIdempotency.mts`. A periodic cleanup is started from `services/idempotencyCleanup.mts` (1 hour cadence, first sweep 30 s after boot).

## Conventions & gotchas

- **Never hand-edit `types.mts`.** It's at `apps/server/src/db/kysely/types.mts` and the first line literally says "Please do not edit it manually." Always regenerate via `pnpm generate-types` after a migration.
- **Migration numbering is mixed 4-digit / 3-digit.** Lexicographic sort still works because `018` > `0017` as strings (the leading zero matters). Use 3 digits for anything new.
- **`bootstrap.mts` does NOT run migrations**. Both calls are commented out (`bootstrap.mts:9-10`). You must `pnpm migrate` manually after pulling new migrations.
- **Trigger retirement is real**: migrations 019/021 are still in the folder, but 026 dropped their triggers + tables. If you bring `envelop_balances` back, you must also restore the triggers OR (preferred) keep the on-read computation pattern.
- **`safeAwait` doesn't preserve stack info nicely**: errors come back as `unknown`-typed (default `E = Error`). When the caller wants finer-grained handling, type the call site (`safeAwait<T, TRPCError>(…)`) or `instanceof TRPCError` checks like `signup/initiate.mts:98`.
- **`SET TIME ZONE` is per-connection, persists across pool checkouts.** If you add a connection-level setting, do it in the same `on("connect")` hook (`db/index.mts:20-39`).
- **`pg.Pool` `ssl: { rejectUnauthorized: false }`** is on for everyone, including the docker-compose Postgres. Production deployments behind Cloudflare/Neon rely on this; do not flip without testing.
- **DEMO mode = pg read-only**: `IS_DEMO` adds `SET default_transaction_read_only = on`. Mutations through tRPC are also blocked separately (`filterMutationsOnDemoMiddleware`). Both layers must be intact.
