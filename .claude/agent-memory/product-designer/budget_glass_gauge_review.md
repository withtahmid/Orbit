On branch `budget-ui-update`, the Budgets page (`apps/web/src/pages/space/budgets/BudgetsPage.tsx`) replaced fill-as-you-spend `ProgressBar`s with an SVG skeuomorphic "fluid glass of money" gauge (`components/budget-gauge/EnvelopeGlass.tsx` + `status.ts`). Spend envelopes DRAIN (liquid = remaining); goals FILL toward target (inverted glass, gold "1× TARGET" line).

**Verdict:** Drain semantics are RIGHT for an envelope (liquid = remaining = the question users ask) — keep them. The mistakes are in execution, not the metaphor.

**Why / how to apply:** Three reusable design judgments came out of this review for the Orbit budgeting surfaces —

1. **Don't let one visual cue carry two meanings.** Here fill-vs-drain direction was made to double as the spend-vs-goal distinction. Subtle, gets muddy in a real mixed grid. Spend-vs-goal must be carried by an explicit structural/text cue (section heading, label, distinct glass shape), not by which way the liquid sits.
2. **Calm state must be a single semantic color, never the envelope's arbitrary hex.** The gauge maps calm→env.color, warn→--warn, over→--expense. Arbitrary hex on the calm rung destroys the health ramp (a reddish envelope looks like a warning). Envelope hex belongs to IDENTITY surfaces (avatar/header), not the health gauge. This is a recurring trap whenever a per-entity color meets a semantic status ramp.
3. **Skeuomorphism clashes with the "editorial-dark" system** (self-described at `apps/web/src/styles/orbit-design.css:5` — flat cards, hairline borders, subtle shadows). Sloshing liquid + sheen + embossed quarter-marks + vessel housing is the loudest object in the app and reads as imported from another product (it was — "bundled budget-gauge design pack"). New rich components must be re-skinned INTO the flat system, not dropped in.

**Other findings:** glass drains while the "% of budget spent" foot rises (same-card direction inversion — flip foot to "% remaining"); under-funded goals show no target marker (gold line only on over-fund); empty-glass + positive-remaining contradiction at the rounding boundary (`hasLiquid = liquidFrac > 0.001`); grid/list flat views interleave spend+goal glasses with no divider while only the grouped view sections them; idle slosh animation is the only perpetual motion in a calm dashboard. List view specifically lost the old bar's comparability — recommended keeping a flat bar there.

**2026-07-01 follow-up (same branch):** User acted on most P0/P1. Resolved: liquid color now SEMANTIC (calm=--brand, warn=--warn, over=--expense, goals=--gold), identity hue only on avatar — goals now read as gold-fill vs spend-drain, so fill/drain no longer solo-carries spend-vs-goal (judgment #1 + #2 fixed). Foot flipped to "% left" (agrees with drain). Goals cap at rim (rescale gimmick dropped for save variant; rescale survives only for spend overspend deficit). "drift" word → "running low" (amber, <warnAt..1 paired text cue, a NON-color signal); overspend keeps red liquid + red card border (`.env-card-over`) + AlertTriangle + "over" + signed negative. Card is portrait/narrow `minmax(196px,1fr)` auto-fill, glass centered as hero. Deliberate user calls — DO NOT relitigate: idle slosh stays ON (user wants flowing liquid); overspend explainer note + cadence "drift" flag REMOVED on purpose. Remaining concerns I still hold: skeuomorphic bottle vs editorial-dark tonal mismatch persists (smaller now, slightly better, still loudest object); `hasLiquid>0.001` rounding boundary still un-fixed (1% remaining shows empty glass while hero says positive); under-funded goal still has no target reference mark in-glass (cap-at-rim makes 40%-saved and 95%-saved hard to tell apart at a glance); flat grid/list still interleave spend+goal with no divider outside grouped view. New compact-layout watch items: Spent/Allocated two-up + long taka at 196px; mobile floor dropped to 150px (very tight for a bottle + two money figures).

**2026-07-01 (round 3) — scope cuts landed.** User DELIBERATELY removed (do NOT relitigate): the "Spent by priority" bar/legend, the "Needs attention" card, the List view AND Grouped view (bottle-card grid is now the ONLY view; view toggle gone), the header subtitle. Overspend gauge reworked: >100% over fills glass red with a "1× budget" line at 1/r height; readout reads "X% over budget". Current composition: topbar (title + Sort + New) → slim summary strip (month nav + Allocated/Spent/Remaining inline + UnbudgetedBanner w/ 90d breakdown) → search → grid. Mid-month nudge toast kept.

Verdict round 3: cuts are sound; page reads as one coherent material now. Genuine remaining issues I flagged: (1) NO sectioning + cadence default sort interleaves goals (gold cards) mid-grid between monthly/rolling — goals should sort LAST by default so the gold band is contiguous (non-toggle fix, honors the no-grouped-view cut). (2) UnbudgetedBanner over-weights the "quiet" summary strip — it has dot+title+sub+overspend pill+expandable 90d breakdown+CTA, louder than the 3 stats above it; demote breakdown to the BudgetMonth page or make the whole banner a single collapsed line. (3) "X% over budget" is honest but at huge over-% (e.g. 1900%) the number is noise — cap display at "999%+ over" or switch to "Nx over". The 1/r line is fine. (4) Goals readable but under-funded goals STILL have no in-glass target reference (old open item persists — 40%-saved vs 95%-saved look similar; gold fill height is the only cue). (5) `/s/me`: Budgets is STRIPPED from PERSONAL_NAV (SpaceLayout.tsx:73 — only "",accounts,transactions,analytics survive), so the page is UNREACHABLE via nav on the virtual space; route still resolves if URL typed → would call analytics.envelopeUtilization with spaceId="me" (wrong router, virtual space owns no envelopes). Low-priority sharp edge: add an isPersonal guard/redirect, or accept it's unreachable. The `hasLiquid = liquidFrac > 0` boundary from round 2 is FIXED (now `> 0` + min-sliver, no more empty-glass-with-positive-hero contradiction).
